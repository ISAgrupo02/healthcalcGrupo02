Refactoring 3

Bad smell / problema:
Primitive Obsession. La clasificación completa del BMI se representaba mediante cadenas de texto, por ejemplo "Normal", "Overweight" u "Obesity Class I". Esto hace que el código dependa de valores escritos manualmente y puede provocar errores si se escribe mal una categoría o si se repite la misma lógica en varios puntos.

Refactorings aplicados:
Se ha aplicado Extract Class creando el nuevo enum BMICategory para representar las categorías del BMI como un tipo propio del dominio. También se ha aplicado Extract Method, separando la lógica de clasificación en el método bmi_category(bmi). Además, se ha aplicado Add Method to Class añadiendo el método category(person), tal y como aparece en el diagrama de la práctica.

Tipo / categoría:
Class refactoring y Method refactoring.

Descripción del cambio:
Se ha creado el enum BMICategory con todas las categorías posibles del BMI. La clase HealthCalcImpl ahora calcula la categoría mediante el método bmi_category, que devuelve un valor de BMICategory en lugar de trabajar directamente con strings. El método bmi_full_classification se mantiene para conservar la compatibilidad con el comportamiento anterior y con los tests existentes.

Cambios manuales:
1 fichero nuevo creado: bmi_category.py.
1 método añadido en HealthCalc: category(person).
3 métodos añadidos o modificados en HealthCalcImpl: bmi_category(bmi), category(person) y bmi_full_classification(bmi).
2 tests nuevos añadidos para comprobar que se devuelve BMICategory.

REFACTORING 4

(1) Bad smell o problema:
Inadecuada descomposición / falta de correspondencia entre el diseño y la implementación. El cálculo del Body Mass Index ya existía como bmi_person(person), pero no había una interfaz específica que representara el concepto BodyMassIndex indicado en el diagrama de la práctica.

(2) Refactoring aplicado:
Extract Interface / Add Method to Class.
Se ha creado la interfaz BodyMassIndex y se ha añadido el método body_mass_index(person), que reutiliza el cálculo BMI ya existente mediante bmi_person(person).

(3) Tipo/categoría:
Class refactoring y Method refactoring.

(4) Descripción:
Se ha creado una nueva interfaz BodyMassIndex para representar el cálculo del índice de masa corporal de una persona. La clase HealthCalcImpl implementa esta interfaz y añade el método body_mass_index(person), que delega en bmi_person(person), manteniendo el mismo comportamiento externo del sistema.

(5) Cambios manuales:
1 fichero nuevo creado: body_mass_index.py.
1 import añadido en HealthCalcImpl.
1 interfaz añadida a la herencia de HealthCalcImpl.
1 método nuevo añadido en HealthCalcImpl: body_mass_index(person).
1 test nuevo añadido en test_bmi.py.
